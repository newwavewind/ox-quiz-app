# OX 변환 규칙

## 문항 유형

| 지문 키워드 | 유형 | 정답 보기 | 오답 보기 |
|------------|------|----------|----------|
| 옳은 것, 해당하는 것, 인정되는 것 | `correct` | O | X |
| 틀린 것, 해당하지 않는 것, 아닌 것 | `wrong` | X | O |
| 모두 고른 것 (복합선택) | `composite` | 정답 조합 보기 O, 나머지 X | |

## 보기 처리

- 보기 ①~⑤ 각각을 독립 OX 문항으로 변환
- 복합형(ㄱㄴㄷ)은 보기 전체 문장을 `statement`로 사용
- 지문에 「다툼이 있으면 판례에 따름」은 statement에 포함하지 않음

## 해설

1. hypers84/수동 CSV 공식 해설 우선
2. 없으면 조문·판례 키워드 기반 짧은 안내 문구 + `needs_review: true`

## 검수

- `needs_review`: 정답 미확보, 복합형, PDF 파싱 불완전
- 연도별 5% 샘플 수동 검수 권장
